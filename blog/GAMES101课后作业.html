Q1 三角形内部判断​

在insideTriangle函数中我们判断点是否在三角形范围内，但是这里值得注意的是，点是一个二维的点，而三角形所给定的是一个三维的点，我们是否应该直接将二维点补足为三维点或者将三位点直接移除z然后进行内外判断？

联系整个project，发现其内部的draw函数中，首先经历了MVP变换后再进行视口变换再进行光栅化，这时对于三维点xyz其实际上已经变换到屏幕坐标系了，xy表示屏幕坐标，因而可以直接利用xy参与检验计算

计算完成后，程序运行发现遮挡部分计算无误，但是对于三角形包围盒中的点是否属于三角形存在问题.

经过查找发现原来是判断逻辑中向量积求解存在问题，应该利用叉积求解方向，而不是点积

static bool insideTriangle(int x, int y, const Vector3f* _v)
{   
    auto vec_cross = [](Vector3f a,Vector3f b){
        a.z() = 0;
        b.z() = 0;
        auto ret = a.cross(b);
        return ret;
    };

    Vector3f p;
    p<< x,y,0;
    auto t1 = vec_cross(p-_v[0],_v[1]-_v[0]);
    auto t2 = vec_cross(p-_v[1],_v[2]-_v[1]);
    auto t3 = vec_cross(p-_v[2],_v[0]-_v[2]);

    if(t1.dot(t2)>0&&t1.dot(t3)>0&&t2.dot(t3)>0) return true;
    return false;
}

在实现的过程中我们首先使用lamda函数求解向量之间的叉积，然后再利用叉积结果两两点乘确定其方向是否一致
