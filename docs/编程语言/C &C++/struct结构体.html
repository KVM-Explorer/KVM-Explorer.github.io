C 语言结构体

组成​

结构体我们认为其是基本数据类型乃至复合数据类型的集合是一种自定义的数据类型可以像int、float等进行定义，其语法组成为struct结构体名称+结构体成员+分号结尾。使用其进行定义的过程也需要附带struct声明。

struct Data1
{
    int a;
    char b;
    float c[10];
    int *x;
};

struct Data2
{
    int a;
    char b;
    struct Data1 data;
};

typedef struct Data3
{
    int a;
    char b;
    struct Data1 data;
}Data3;

typedef struct Data4
{
    int c;
}Data4,*Data4Ptr;

struct Data1 tmp;
Data3 tmp2;
struct Data1 data1_array[10];

为了简化书写，C语言提供了typedef关键字帮助我们实现类型的重命名，这里需要注意的是结构体指针的重命名，对于结构体指针我们需要搞清楚其类型。以Data4为例其指针指的是将Data4 *重名为Data4Ptr

内存​

为了方便理解和便于自检，强烈推荐Vscode的clangd插件可以直接查看内存排列情况

Struct结构体实际上是内存中一段连续的内存，其内部元素按照定义顺序依次排列，其理解难点主要在于两点：结构体整体内存的理解和结构体内存对齐。

基本理解​

首先要理解计算机中的内存的组织方式，虽然其底层可能是连续或者离散的，但是经过操作系统的屏蔽从逻辑上后我们认为其是一段连续的线性内存。在这个基础上我们的结构体就是内存上的一段连续空间，其内部的元素按照其定义的顺序并根据内存对齐的方式依次排列在内存中。

内存对齐​

在设计不同CPU下的通信协议时，或者编写硬件驱动程序时（硬件具备寄存器和自己的处理器）两个地方都需要按规定数目字节对齐（如GPU和CPU、CPU和打印机等外设）。即使看起来本来就自然对齐的也要使其对齐，以免不同的编译器生成的代码不一样。

除了数据通信，对于计算机而言，字节对齐可以使内存访问的速度大大增加。以32位处理器为例，其工作原理为每次取32位长度的指令，对指令的内容进行解析，提取其中的数据存储的寄存器和指令信息。倘若，其指令是非内存对齐可能CPU需要根据情况进行查找和定位真正的指令位置，进而可能需要多个时钟周期处理同一条指令。

内存对齐在设定对齐字节的情况下（这个由编译器决定，可进行自定义修改，）主要遵循如下两条原则：

  1. 结构体每个成员相对于起始地址的偏移能够被其自身大小整除，如果不能则在前一个成员后面补充字节
  2. 结构体总体大小能够被最宽的成员的大小整除，如不能则在后面补充字节
struct MemoryAlignment1 // 8 bytes
{
    char ch;  // 1 + 3 Padding
    int data; // 4
};

struct MemoryAlignment2 // 8 bytes
{
    int data; // 4
    char ch;  // 1
    char ez;  // 1 + 2 Padding
};

结构体成员的内存​

对于结构体成员而言其按照其定义顺序依次排列在结构体所在的内存中，因而对于结构体成员我们往往采用偏移offset描述其开始地址并根据其类型和内存对齐原则判断其所在内存。

结构体地址​

对于一个结构体而言，其地址即结构体整体内存的开始地址。

内存中的大小端​

不仅仅是结构体，实际上任何计算机存储的数据都会面临大端小端的问题。首先我们来认识一下什么是大端小端？大端小端指的是数据的高位在内存中存储地址的位置。对于大端存储，其数据的高位在低地址，而对于小端存储，其数据的高位在高地址。举个简单的例子：首先假定内存地址从左往右依次递增，对于数字1234，其在内存中如果是大端存储那么其在内存中是按照1234进行存储，而小端存储则是4321 。当然计算机的是由01构成，而对于一个计算机系统而言其往往以字为最小计量单位（可能1个字是1字节或者2字节等），那么对应大小端也是以字为单位进行划分，这里以1字1字节下的int类型为例。int占据了4个字节

// 地址 0 - > 
00000000 00000000 10000000 10001000 // 十进制下32904 大端存储
10001000 10000000 00000000 00000000 // 十进制下32904 小端存储

结构体初始化​

默认​

结构体在定义的过程中面临着和普通变量相同的问题，即全局变量默认情况下初始化为默认值，而局部结构体变量内部的值由分配的内存决定（因而处于随机状态）

方法一 初始化列表​

struct Init
{
    int data;
    char ch;    
};
struct Init a = {10,'c'};

方法二 逐个初始化​

struct Init
{
    int data;
    char ch;    
};
struct Init b;

a.data = 10;
a.ch = 'c';

方法三 乱序初始化​

struct Init
{
    int data;
    char ch;    
};

struct Init c = {
    .ch = 'd',
    .data = 10
};

应用​

基本应用​

结构体主要发挥数据封装的作用，将同类型或者同用途的数据进行封装。

// 普通结构体
struct Sample1
{
    int data;
    char ch;
}sample1_instance;

// 结构体指针
struct Sample1 *sample1_ptr;

int main()
{
    int data = sample1_instance.data;

    sample1_ptr = &sample1_instance;
    char ch = sample1_ptr->ch;
}
