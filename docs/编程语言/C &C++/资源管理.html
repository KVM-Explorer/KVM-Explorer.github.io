资源管理RAII

getter/setter装饰器​

在资源管理中，如果涉及修改的资源会引起其他数据的变化或更改，这时我们采用getter/setter装饰器进行修饰，反之我们应该采用public直接暴露对应成员变量。

一方面，在高性能计算中getter/setter装饰器在频繁调用函数的过程中性能开销不可忽视，特别是在函数声明和实现分离在不同的文件中。另一方面，装饰器增大了开发实现成本和用户的学习使用成本。

智能指针​

在C++11后新增了智能指针大大方便了对于内存资源的管理，同时根据其不同的应用场景分为了三种类型

  • unique_ptr

  • shared_ptr

  • weak_ptr

shared_ptr​

shared_ptr提供了引用计数功能，当资源被引用时引用计数++，反之--。当引用计数归零，则自动释放资源，这就是其内在的内存管理方式。那么这种方式有没有什么问题呢？面对一般的应用场景其是足以用于内存管理，但是在一些复杂的情况下

  • 链表

    • 循环引用

  • 多线程下对引用对象的操作

多个线程之间，递增和减少资源的引用计数是安全的。（注意：这不意味着多个线程同时操作 std::shared_ptr 引用的对象是安全的）。
