多页面导航和数据刷新​

Frame 不会在导航历史记录中存储内容对象的实例。 相反， Frame 每次使用导航历史记录导航到内容对象时，都将创建该对象的新实例。 此行为旨在避免在将大量内容导航到时消耗过多的内存。因此，不会从一个导航到下一个导航来记住内容的状态。不过，WPF 提供了多种方法，你可以通过这些方法在导航历史记录中存储部分内容的状态。

Frame 通过Navigate导航到不同的内容时，每次都会创建新的对象实例。这种方法可能发生一些问题，如在A界面不断更新Image控件时，切换到B界面。一段时间切回A界面，此刻后台绑定的Image控件仍属于旧的对象，因而无法正常更新，甚至可能出现跨线程依赖导致的崩溃问题。

  1. 采用容器提前创捷不同的界面Object进行存储,如果直接采用Navigate->Uri的话是根据Uri在Navigation函数内部创建Page
  2. 更改UI和后台数据的交互模式，采用UI拉取数据的单向数据绑定以解耦合后台对于前台相同对象不同示例的错误绑定

方法一自定义XAML Style​

通过自定义控件的Style来修改控件的显示属性，这里以Button的Background属性为例 前台XAML配置，由于整体是基于MaterialDesign主题，因而这里Style配置的过程中使用了BasedOn属性，否则可以省略完全自定义Button样式，其关键在于通过DataTrigger判断绑定数据的值，进而帮助判断启用何种样式

<Button Tag="{Binding}" Content="Test"">
    <Button.Style>
        <Style TargetType="Button"
                BasedOn="{StaticResource MaterialDesignFlatButton}">
            <Setter Property="Background" Value="Transparent" />
            <Style.Triggers>
                <DataTrigger Binding="{Binding IsFullFlag}" Value="true">
                    <Setter Property="Background" Value="Khaki" />
                </DataTrigger>
                <DataTrigger Binding="{Binding IsFullFlag}" Value="false">
                    <Setter Property="Background" Value="WhiteSmoke" />
                </DataTrigger>
            </Style.Triggers>
        </Style>
    </Button.Style>
</Button>

后台数据绑定（非Dependence Page），在已有窗口实例MainWindowInstance的条件下，使用Dispatcher.invoke异步更新前端UI数据

// 后台属性get;set;配置属性
private bool _IsFullFlag = false;
public bool IsFullFlag
{
    get => _IsFullFlag;
    set{
        _IsFullFlag = value;
        updateProperty(nameof(IsFullFlag));
    }
}
// 后台通知前台更新
private void UpdateProperty(string propertyName)
{
    MainWindowInstance.Dispatcher.Invoke((Action) (() => // 当前窗口实例instance
    {
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
    }));
}

WPF后台（非Dependence Page），在后台拥有窗口实例MainWindowsInstance的条件下，使用Dispatch.invoke异步更新

// 后台属性get;set;配置属性
private SoildColorBrush _background;
public SolidColorBrush background{
    get=>_background;
    set{
        _background = value;
        updateProperty(nameof(background));
    }
}

//  根据后台条件更新属性
if( <condition> )
{
    background = new SoildColorBrush(Colors.Black);
}else
{
    background = new SoildColorBrush(Colors.White);
}

// 后台通知前台更新
